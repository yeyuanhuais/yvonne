 <Highlight>信息系统项目开发的可行性包含可能性、效益性和必要性三个方面，三者相辅相成，缺一不可</Highlight> 

## ★ 2.1. 可行性研究的内容
1. 技术可行性分析：指在当前市场的技术、产品条件限制下，能否利用现在拥有的以及可能拥有的技术能力、产品功能、人力资源来实现项目的目标、功能、性能，能否在规定的时间期限内完成整个项目。
    1. 进行项目开发的风险
    2. 人力资源的有效性
    3. 技术能力的可能性
    4. 物资（产品）的可用性
    5.  <Highlight>技术可行性分析往往决定项目的方向，一旦开发人员在评估技术可行性分析时估计错误，将会出现严重的后果，造成项目根本上的失败</Highlight> 
2. 经济可行性分析： <Highlight>主要是对整个项目的投资及所产生的经济效益进行分析，具体还包括支出分析、收益分析、投资回报分析以及敏感性分析等</Highlight> 
    1. 支出分析：
        1.  <Highlight>一次性支出：开发费、培训费、差旅费、初始数据录入、设备购置费等费用</Highlight> 
        2.  <Highlight>非一次性支出：软硬件租金、人员工资及福利、水电等公用设施适用费，以及其他消耗品支出等</Highlight> 
    2. 收益分析：
        1. 直接收益：如销售项目产品的收入
        2. 间接收益：如成本降低
        3. 其他收益
    3. 收益投资比、投资回收期分析：对投入产出进行对比分析，以确定项目的收益率和投资回收期等经济指标
    4. 敏感性分析：当诸如设备和软件配置、处理速度要求、系统的工作负荷类型和负荷量等关键性因素变化时，对支出和收益产生影响的估计
    5.  <Highlight>社会效益分析：如通过项目的实施，可以在管理水平、技术手段、人员素质等方面获得潜在的效益</Highlight> 
3. 运行环境可行性分析
    1.  <Highlight>运行环境是制约信息系统在用户单位发挥效益的关键</Highlight> 
4. 其他方面的可行性分析
    1. 诸如法律可行性、社会可行性等方面
    2. 合同责任、知识产权等法律方面的可行性问题

## 2.2. 可行性研究步骤
1. 确定项目规模和目标
2. 研究正在运行的系统
3. 建立新系统的逻辑模型
4. 导出和评价各种方案
5. 推荐可行性方案
6. 编写可行性研究报告
7. 递交可行性研究报告

:::note
可行性研究分为初步可行性研究、详细可行性研究、可行性研究报告

在初步项目可行性研究之前可进行项目机会研究，如果就投资可能性已进行了项目机会研究，那么项目的初步可行性研究阶段往往可以省去。
:::

## 2.3. 初步可行性研究

### 2.3.1. 初步可行性研究的定义及目的
定义：一般是在对市场或者客户情况进行调查后，对项目进行的初步评估

形成初步可行性研究报告

### 2.3.2. 初步可行性评估内容
1. 分析项目的前途，从而决定是否应该继续深入调查研究
2. 初步估计和确定项目中的关键技术及核心问题，以确定是否需要解决
3. 初步估计必须进行的辅助研究，以解决项目的核心问题，并判断是否具备必要的技术、实验、人力条件作为支持

### 2.3.3. 初步可行性研究回答的问题
1. 项目进行投资建设的必要性
2. 项目建设的周期
3. 项目需要的人力、财力资源
4. 项目的功能和目标是否可以实现
5. 项目的经济效益、社会效益是否可以保证
6. 项目从经济上、技术上是否合理

### 2.3.4. 初步可行性研究的主要内容
1. 市场和生产能力：进行市场需求分析预测，渠道与推销分析，初步的销售量和销售价格预测；依据市场销量做出初步开发规划
2. 设备与材料投入分析：从需求、设计、开发、安装实施到运营的所有设备与材料的投入分析
3. 网络规划、物理布局方案的选择
4. 项目设计包括项目总体规划、信息系统设计和设备计划、网络工程规划等
5. 项目进度安排
6. 项目投资与成本估算，包括投资估算、成本估算、筹集资金的渠道及初步筹集方案

### ★ 2.3.6. 辅助（功能）研究

辅助研究的内容视研究的性质和打算研究的项目各有不同，但由于其关系到项目的关键方面， 因此其结论应为随后的项目阶段指明方向。 在大多数情况下， 投资前辅助研究如果在项目可行性研究之前或与项目可行性研究起进行，其内容则构成项目可行性研究的一个必不可少的部分。

辅助研究的费用必须和项目可行性研究的费用联系起来，这种研究的一个目的是要在项目可行性研究阶段节省费用

:::note
辅助研究包括项目的一个或几个方面，但不是所有方面

并且只能作为初步项目可行性研究、项目可行性研究和大规模投资建议的前提或辅助
:::

### 2.4.4. 详细可行性研究的方法
1. 经济评价法
2. 市场预测法
3. 投资估算法
    1. 指数估算法：$x=y(c_{2}/c_{1})^{0.6} C_{F}$
        1. $x$——投资估算数
        2. $y$——同类老项目的实际投资数
        3. $c_{2}$——新项目的生产能力
        4. $c_{1}$——老项目的生产能力
        5. $C_{F}$——价格调整系数
    2. 因子估算法：因子是指主要设备与其他设备的比值
    3. 单位能力投资估算法：$K=aQ$
        1. $K$——被估算项目投资额
        2. $a$——单位生产能力投资额
        3. $Q$——被估算项目的生产能力
4. 增量净效益法（有无比较法）：将有项目时的成本（效益）与无项目时的成本（效益）进行比较，求得两者差额，即为增量成本（效益），这种方法称之为有无比较法

### 2.4.5. 详细可行性研究的内容
1. 概述：提出项目开发的背景、必要性和经济意义，研究项目工作的依据和范围，产品交付的形式、种类、数量
2. 需求确定：调查研究国内外客户的需求情况，对国内外的技术趋势进行分析，确定项目的规模、目标、产品、方案和发展方向
3. 现有资源、设施情况分析：调查现有的资源（包括硬件设备、软件系统、数据、规章制度等种类与数量，以及这些资源的使用情况和可能的更新情况）
4. 设计（初步）技术方案：确定项目的总体和详细目标、范围，总体的结构和组成，核心技术和关键问题、产品的功能与性能
5. 项目实施进度计划建议
6. 投资估算和资金筹措计划
7. 项目组织、人力资源、技术培训计划：包括现有人员规模、组织结构、人员层次、个人技术能力、人员技术培训计划等
8. 经济和社会效益分析（效果评价）
9. 合作/协作方式

:::tip
开发总成本一般划分为四大类：研发成本、行政管理费、销售与分销费用、财务费用和折旧
:::
