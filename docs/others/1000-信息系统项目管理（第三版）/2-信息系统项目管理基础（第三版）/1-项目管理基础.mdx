## 1.1. 项目的概念
项目是为提供一项独特产品、服务或成果所做的临时性努力

## ★ 1.2. 项目特点
1. 临时性（<Highlight>一次性</Highlight>）
    1. 是指每一个项目都有确定的<Highlight>开始和结束日期</Highlight>，当项目的目的已经达到，或者已经清楚的看到该目的不会或不可能达到时，或者该项目的必要性已不复存在并已终止时，该项目即达到了它的终点。
    2. 临时性一般不适用于项目所产生的产品、服务或成果。大多数项目是为了得到持久的结果。
2. 独特的产品、服务或成果
    1. 项目<Highlight>创造独特的可交付成果</Highlight>，如产品、服务或成果。
    2. 独特性是项目可交付成果的一种重要特征。
3. 逐步完善
    1. 逐步完善是项目伴随临时性和独步性两个概念的特点之一。逐步完善意味着<Highlight>分步、连续的积累</Highlight>。
    2. 例如：在项目的早期，项目范围的说明是粗略的，随着项目团队对目标和可交付成果的理解更完整和深入时，项目的范围也就更具体和详细
4. 资源约束
    1. 每个项目都需要具备各种资源来作为实施的保证，而资源是有限的。所以，资源成本是项目成功实施的一个约束条件。
5. 目的性
    1. <Highlight>项目工作的目的在于得到特定的结果，即项目是面向目标的</Highlight>
    2. 时间、成本和质量是项目工作的三个主要目标，有时候被称为项目成功的三约束。另一个主要约束时项目的范围
    3. 项目经理的责任就是在时间、成本、质量、项目范围这四个要素之间进行权衡以保证项目成功

## 1.3. 项目与日常运作的关系
1. 相同之处：由人来做，受制于有限的资源，需要规划、执行和控制
2. 区别：
    1. 日常运作是持续不断和重复进行的，而项目是临时性的、独特的
    2. 项目的目标是实现其目标，然后结束项目，而持续进行的日常运作的目标一般是为了维持经营
    3. 项目的实现机制与日常运作大相径庭，因为当宣布的目标实现时，项目就结束了，相比之下，日常运作是确定一组新目标，然后持续进行

| 不同点   | 项目         | 日常运作       |
| -------- | ------------ | -------------- |
| 目的     | 独特性       | 常规的、普遍的 |
| 责任人   | 项目经理     | 部门经理       |
| 持续时间 | 有限的       | 相对无限的     |
| 持续性   | 一次性       | 重复性         |
| 组织结构 | 项目组织     | 职能部门       |
| 考核指标 | 以目标为导向 | 效率和有效性   |
| 资源需求 | 多变性       | 稳定性         |

## 1.4. 项目和战略规划
企业战略：是对企业各种战略的统称，其中既包括竞争战略、营销战略、发展战略、品牌战略、融资战略、技术开发战略、人才开发战略、资源开发战略等等。

:::tip
企业战略虽然有很多种，但基本属性是相同的，都是对企业的谋略，都是对企业整体性、长期性、基本性问题的计谋

不是根据企业当前经营状况所制定
:::

### ★ 1.4.1. 战略管理
1. 战略制定：确定企业任务，认定企业的外部机会与威胁，认定企业内部优势与弱点，建立长期目标，制定供选择战略，以及选择特定的实施战略
2. 战略实施：树立年度目标、制定政策、激励员工和配置资源，以便制定的战略得以贯彻执行
3. 战略评价：重新审视外部与内部因素，度量业绩，采取纠偏措施

### 1.4.2. 项目批准的典型依据
1. 市场需求
2. 运营需要
3. 客户要求
4. 技术进步
5. 法律要求

## 1.5. 信息系统项目的特点

:::note
信息系统是以信息的集成为目标，功能的集成为结构，平台的集成为基础，人的基础为保证
:::

1. 目标不明确
2. 需求变化频繁
3. 智力密集型
4. 设计队伍庞大
5. 设计人员高度专业化
6. 涉及的承包商多
7. 各级承包商分布在各地，相互联系复杂
8. 系统集成项目中需研制开发大量的软硬件系统
9. 项目生命期通常较短
10. 使用与维护的要求非常复杂

## 1.6. 项目管理定义
项目管理就是把各种知识、技能、手段和技术应用于项目活动之中，以达到项目的要求

项目管理是通过应用和综合诸如启动、计划、实施、监控和收尾等项目管理过程来进行的

管理一个项目包括：识别要求；确定清楚而又能够实现的目标；权衡质量、范围、时间和成本方面互不相让的要求；使技术规格说明书、计划和方法适合于各种各样项目干系人的不同需求于期望等内容

### 1.6.1. 项目管理的理解
1. 项目管理是一种管理方法体系。是一种已被公认的管理模式，而不是任意的一次管理过程
    1. 项目管理作为一种管理方法体系， 在不间国家、 不同行业以及它自身的不同发展阶段，无论在结构、 内容L 还是在技术、 手段上都有一定的区别。 但它最基本的方面，就是上述定义中所规定的那些内容， 则始终如一，相对固定，且已形成为一种被公认的专业知识。
1. 项目管理的对象、目的。
    1. 对象是项目，即一系列的临时任务。项目是由一系列任务组成的整体系统，而不是整个整体的一个部分或几个部分
    2. 目的是通过运用科学的项目管理技术，更好的实现项目目标
    3. 不能把项目管理对象与企业管理的对象混为一谈，项目只是企业庞大系统的一部分；也不能把企业管理的目的当成项目管理的目的，企业管理的目的是多方面的，而项目管理的主要目的是实现项目的预定目标
2. 项目管理的任务、职能
    1. 职能是对组织的资源进行计划、组织、指挥、协调、控制
    2. 任务是对项目及其资源的计划、组织、指挥、协同、控制
3. 项目管理运用系统理论与思想
    1. 项目在实施过程中，实现项目目标的责任和权力往往被集中到一个人或一个小组身上
4. 项目管理的职能主要是由项目经理执行的

## 1.7. 项目管理的特点
1. 项目管理是一项复杂的工作
2. 项目管理具有<Highlight>创造性</Highlight>：创新可以是技术创新、管理方法创新、流程创新
3. 项目管理需要集权领导和建立专门的项目组织
4. 项目负责人在项目管理中起着非常重要的作用：成功的管理还取决于预测和控制人的行为的能力，项目负责人必须使他的组织成为一只真正的队伍，一个工作配合默契、具有积极性和责任心的高效率群体
5. 社会经济、政治、文化、自然环境等对项目的影响
