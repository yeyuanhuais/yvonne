### 请解释一下微前端的概念以及它的主要优点和挑战?
优点：
1. 解耦：分为多个独立开发、测试、部署的应用。提高开发效率，减少团队间协调成本
2. 技术栈无关：支持不同的技术栈，可以使用新技术、升级旧技术
3. 并行开发：互不影响
4. 独立部署：
挑战：
1. 性能问题：使用不同的库或框架，可能会导致加载和运行的性能问题
2. 一致性：用户体验、设计和行为上的一致性会比较困难
3. 状态共享：需要使用特殊的工具或模式实现
4. 复杂性：比如需要管理和协调多个独立应用
5. 安全性：增加跨域等安全问题

### 你能详细描述一下 qiankun 微前端框架的工作原理吗？
基于single-spa的微前端实现框架
1. 应用加载：通过动态创建script标签方式加载子应用的入口文件。加载完成后执行子应用暴露出的生命周期函数
2. 生命周期管理：子应用需要暴露出bootstrap、mount、unmount。bootstrap在应用加载时被调用，mount应用启动时被调用，unmount应用卸载时被调用
3. 沙箱隔离：通过proxy对象创建一个js沙箱，用于隔离子应用的全局变量，防止子应用之间的全局变量污染
4. 样式隔离：动态添加和移除样式标签的方式实现样式隔离，应用启动就添加，卸载就移除
5. 通信机制：全局通信机制，允许子应用之间进行通信
